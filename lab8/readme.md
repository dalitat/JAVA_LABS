Лабораторная работа №8

В данной лабораторной работе изучаются механизмы аннотаций, Stream API
и многопоточности в Java.

Создана собственная аннотация @DataProcessor с политикой удержания
RetentionPolicy.RUNTIME и целевым элементом ElementType.METHOD.

Реализован класс FileManager для загрузки данных из файла и сохранения
результатов с использованием Files.readAllLines() и Files.write().

Разработан класс DataManager, который управляет многопоточной обработкой
данных: регистрирует объекты-обработчики через registerDataProcessor(),
загружает данные через loadData(), обрабатывает их через processData()
с использованием ExecutorService и Future, а также сохраняет результаты
через saveData().

Созданы три класса-обработчика: FilterProcessor с методом filter(),
который с помощью Stream API оставляет только строки длиной более 4
символов; TransformProcessor с методом transform(), переводящим все
строки в верхний регистр; AggregateProcessor с методом count(),
выводящим количество элементов.

В классе Main демонстрируется работа приложения: регистрация обработчиков,
загрузка из input.txt, обработка и сохранение в output.txt. Входные данные:
amamam, denal, o, cat, banana, dog, elephant, tiger. После фильтрации
остаются строки длиной более 4 символов, затем они преобразуются в
верхний регистр. Количество элементов после обработки — 5.

Все обработчики выполняются параллельно с использованием многопоточности.
В результате освоены создание аннотаций, применение Stream API для
функциональной обработки данных и использование многопоточности для
параллельного выполнения задач.
